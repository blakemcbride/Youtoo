# Youtoo - EuLisp System

Youtoo is a public domain implementation of the EuLisp programming language. It compiles EuLisp source to C-embedded bytecodes that are statically or dynamically linked with the Eutopia virtual machine. Originally by Andreas Kind & Julian Padget (University of Bath, 1997), maintained by Henry G. Weller.

See [BUILD.md](BUILD.md) for prerequisites, build steps, and how to run.

## Project Structure

```
Vm/           Virtual machine (Eutopia) - C bytecode interpreter
Telos/        The EuLisp Object System (meta-object protocol)
Runtime/      Level-0 and level-1 standard libraries
Comptime2/    Compiler (EuLysses) - EuLisp → C bytecode
Youtoo/       Main executable entry point
Test/         Test suite (test1.em - test4.em)
Tools/        Build utilities (b2h, i2c, etc.)
include/      C headers for VM and FFI (eulisp.h)
Lib/          Pre-compiled library interface files (.i)
EuLisp/       Language specification and documentation
```

Each subsystem has a `u2/` subdirectory for generated/platform-specific code and an `Ff/` subdirectory for foreign function interface bindings. Architecture-specific code is in `32bit/` and `64bit/` subdirectories.

## Languages

- **EuLisp** (`.em` files): Primary source language. Module-based with `(defmodule name ...)` structure.
- **C** (`.c/.h` files): VM, runtime, and generated bytecode output. Must be C99.
- **Makefiles**: GNU Make with architecture-conditional compilation.

## Code Conventions

### EuLisp
- Modules: `(defmodule name (syntax (syntax-1) import (level-1)) ...)`
- Naming: lowercase-with-hyphens (`list-size`, `make-vector`)
- Classes: angle brackets (`<list>`, `<null>`, `<object>`)
- Predicates: `?` suffix (`null?`, `cons?`)
- Bootstrap files: underscore prefix `_*0.em`
- Comments: `;;;` section headers, `;;` inline

### C
- Function prefixes: `eul_` for public API, `fprint_` for printers
- Types: `LispRef`, `RegisterRef` (Ref suffix for typedefs)
- Macros: `UPPER_CASE` with underscores
- Headers use `#ifndef` include guards
- Copyright headers with `///` style

## Architecture

**Compilation pipeline**: EuLisp source → Compiler (Comptime2) → C bytecode → C compiler → linked with VM

**Build order**: Vm → Telos → Runtime → Comptime2 → Youtoo executable

**Key libraries produced**:
- `libeulvm.a` - Virtual machine
- `libboot.a`, `libtelos.a` - Object system
- `liblevel-0.a`, `liblevel-1.a` - Language levels
- `libmath.a` - Math functions
- `libeval.a`, `libmain.a` - Compiler/evaluator

## Key Files

- `include/eulisp.h` - FFI C API (data conversion between C and EuLisp)
- `include/config.h` - VM configuration (word length, GC, debugging)
- `Vm/interpret.c` - Main VM interpreter loop
- `Runtime/level-1.em` - Level-1 library definition
- `Comptime2/cg-bycode*.em` - Bytecode generation
- `Telos/boot.em` - Object system bootstrap
